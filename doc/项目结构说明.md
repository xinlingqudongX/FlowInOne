# 项目结构说明

##核心模块

### 1. Project 模块 (项目管理)
- **project.entity.ts**: 项目基本信息实体
  - 项目ID、名称、描述
  -基础路径、技术栈配置
  - 创建时间和更新时间
  - 关联工作流图集合

- **project-asset.entity.ts**: 项目资产实体
  -资产ID、文件路径
  - 文件角色分类
  - 文件哈希值用于完整性校验
  - 与项目的一对多关系

### 2. Workflow Graph 模块 (工作流图)
- **workflow-graph.entity.ts**:工作流图实体
  - 图ID、版本号、状态
  -全局状态存储(黑板数据)
  - 与项目和任务节点的关联关系

### 3. Task Node 模块 (任务节点)
- **task-node.entity.ts**: 任务节点实体
  -ID、类型、状态
  - 依赖关系和路由映射
  -指令集(指南、逻辑、标准)
  - RAG增强和上下文信息
  -资产约束和预期产出
  - AI输出和错误日志

- **task-output.entity.ts**: 任务产出实体
  - 产出ID、文件路径
  -操作类型(CREATE/UPDATE/DELETE)
  -差摘要
  - 与任务节点的关联关系

## 数据关系

```
Project (1) ←→ (N) WorkflowGraph (1) ←→ (N) TaskNode (1) ←→ (N) TaskOutput
     │
    └── (N) ProjectAsset
```

##技术特性

- 使用MikroORM作为ORM框架
- SQLite作为默认数据库
-支持JSON字段存储复杂数据结构
-完整的实体关系映射
- 类型安全的TypeScript定义
